2. VERIFY DATABASE TABLES
   - Ensure these tables exist in your Supabase:
     • dms_dealers (with document_pan, document_gst, etc.)
     • dms_inventory_items
     • dms_sales
     • dms_sale_items
     • dms_service_invoices
     • dms_service_invoice_items
   - If missing, create them following the schema in CRM_INTEGRATION_GUIDE.md

3. TEST THE IMPLEMENTATION
   - Run: pnpm dev
   - Navigate to: http://localhost:8080/dealers
   - Click "Dealer Audit" tab (it's the default)
   - Select a dealer from the dropdown
   - Verify data loads correctly

TROUBLESHOOTING QUICK REFERENCE
================================

Problem: Dealer dropdown is empty
→ Solution: Check dms_dealers table exists with data

Problem: No data after selecting dealer
→ Solution: Check if selected dealer has records in related tables

Problem: Documents won't preview
→ Solution: Verify documents are base64 encoded in database

Problem: Download button doesn't work
→ Solution: Check invoice has required fields
